### CORE CONCEPT
RTRM takes the internal activation states from a trained feedforward neural network and attempts to reconstruct the original input data from those activations. By visualizing these reconstructions at each layer, the SME can see:
1. How the network progressively transforms their familiar input data
2. At which layer(s) important information is discarded
3. Whether the network's internal groupings make sense

### THE THREE RTRM METHODS CURRENTLY USED

**Method 1: Cosine Similarity (the Brute-Force method)**
- After training, take a subset of the training examples and record their progression through the network in the form of activation values for various layers or group of neurons.
- Pick strategic "probe points" for closer examination
- At each layer, measure cosine similarity between the probe's activation at that layer and all other subset points' activations. Don't forget to normalize the activations. Apply L2 normalization to activation vectors (for the sub-set and for the probe) before cosine similarity dot product calculations.
- Points with high cosine similarity (e.g., >= 0.9) form an "equivalence class"
- Present those points alongside the probe point in a format familiar to the SME, and indicate at what layer this equivalent class measure was taken. 
- REVEALS: Which subset examples does the network treat as similar to a particular probe point at each layer?
- USES: Identify potential problem areas where the network groups together examples the SME would have considered different, or separates examples the SME might consider similar.

**Method 2: Pseudo-Inverse Reconstruction (also called the Linear Inverse or "Patch" method)**
- Build "patches" by propagating weight matrices backward through the network without using biases or activation functions.
- For the first layer, the "patches" are the weights to the robot's input vector.
- For subsequent layers, if neuron X is connected to a group of prior-layer neurons simply take the "patches" from those prior layer neurons, multiply each by the connection weight to X, add them up, and that is the patch for neuron X. Simplistic justification: repeated successful presentations of that "patch" must have existed in the training data to cause those weights to conform to that "shape."
- Patches need only be created once, but pseudo-inverse takes place for each probe point. The activations at each layer are calculated for that probe point. If it is the first layer, we take the activations, and multiply them by the "patches" (which are the weights) and add them up to create the input vector reconstruction. Subsequent layers use the same math, multiplying the activation at that layer by each patch and then adding them up to reconstruct the input. 
- This is a linear, structural pseudo-inverse
- REVEALS: How much spatial/structural information survives (or is lost due to bottlenecks, pooling, non-linear activations, etc.) within the network's linear transformations
- USES: Identify layers where linear invertibility breaks down, showing where nonlinear transformations dominate. When patch method used on a sub-set of training data, it shows good input recreation in early layers, and then degenerating into grouped "blobs" representing the classification or decision is made. Nevertheless, input reconstruction information within class blobs may still be preserved, and the input might be better reconstructed by other methods. So this method really shows where information is discarded prior to non-linearity taking over.

**Method 3: Learned Inverse (Autoencoder)**
- Train a separate decoder network (per layer) to map activations back to original inputs
- This learns a nonlinear pseudo-inverse
- ARCHITECTURE GUIDELINES:
  * Don't need to recreate at every layer. Select important layers at each major architectural part of the AI solution, but don't skip layers because they seem difficult to solve. The autoencoder will fail to recreate a good estimate of the input at some layer.
  * Final output layer activation is linear (prior to activation function, thresholding, selection from SoftMax, or any other post-processing) since we need to preserve what data we can
  * For very low-dimensional bottlenecks (≤8D): Use deeper decoder networks (4+ layers) with higher capacity
  * Adaptive training: Use lower learning rates and more epochs for challenging low-dimensional layers
- REVEALS: How much information is preserved in the activations, even if not linearly accessible. When autoencoder reconstructions degrade, either the autoencoder has been built incorrectly, or that layer has thrown away information permanently.
- USES: Compare to patch method - if autoencoder succeeds where patch fails, information still exists but is encoded nonlinearly. If autoencoder also fails, information is truly lost. The layer where autoencoder quality drops sharply is where an SME will notice critical information is discarded, they can no longer discern what the input was nor how it should have been classified, or perhaps equivalently where the AI "decision is made" and remaining info is discarded beyond this layer.


### IMPLEMENTATION REQUIREMENTS

When implementing RTRM for a programmer, include:

1. **Probe Point Selection**: Strategic points (class centers, decision boundaries, edges of clusters, outliers)

2. **Per-Layer Analysis**: Capture activations at every layer for all three methods

3. **Presentation Priority**: Recreated input data should be presented in the same format the SME is accustomed to. If it's audio, play back the audio. If it's a chemical notation of a molecule, present it that way. If it's an image or a video, display that. It may be tempting to use a sub-set of the training data for mean and variance and to normalize a batch of reconstructions presented to the SME, but that depends on the particular implementation, and is usually not necessary.
   - Original data (establish baseline)
   - Reconstruction progressions showing degradation through layers
   - Cosine equivalence clusters (who groups with whom)
   - Quantitative error metrics
   - Side-by-side method comparisons per layer

4. **Autoencoder Strategy**: Use deeper networks (3-4 hidden layers) for low-dimensional bottleneck layers; adjust learning rate and training epochs based on layer dimensionality. Optional Joint Training Strategy used to save time if training a new AI solution:
   - Train classification and reconstruction simultaneously
   - Total loss = classification_loss + α × reconstruction_loss
   - Typical α (reconstruction weight) = 0.5
   - Reconstruction provides regularization and encourages invertible representations
   - Use MSE loss for reconstruction: F.mse_loss(reconstructed, original)

5. **Pedagogical Ordering**: Present visualizations in a sequence that builds understanding:
   - Start with familiar (original data)
   - Show transformation (reconstructions)
   - Reveal grouping behavior (cosine equivalence)
   - Quantify loss (error metrics)
   - Compare methods (side-by-side)

6. **Key Metrics to Track**:
   - Reconstruction MSE at each layer (lower is better)
   - Equivalence class sizes (when do they stabilize?)
   - Classification accuracy (does RTRM help performance?)
   - Watch for layers where MSE spikes—these are bottlenecks


### KEY INSIGHTS TO COMMUNICATE

The three methods are **complementary**:
- **Cosine equivalence** shows WHEN decisions are made (equivalence classes stabilize)
- **Patch method** shows WHERE geometric/linear structure breaks down
- **Autoencoder** shows WHAT information still exists even if structure is broken

Different methods may show "decision commitment" (tight clustering/separation) at different layers:
- Early clustering in one method but not others means the decision is made but not yet cleanly encoded
- Divergence between methods reveals architectural inefficiencies or opportunities for improvement
